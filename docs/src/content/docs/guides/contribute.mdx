title: Contribute
description:
  pls is a free and open-source project that needs your contributions in code,
  docs, sponsorships and evangelism.
---

<Pls /> accepts contributions in many forms, ranging from direct code
contributions to sponsorships.

- **Reporting bugs**

  Reporting bugs helps make <Pls /> more stable and reliable. If you find a bug,
  or if something breaks, please
  [open an issue](https://github.com/pls-rs/pls/issues/).

- **Suggesting features**

  <Pls /> aims to be a capable alternative to `ls(1)`. If a feature you need is
  absent from <Pls />, please [open an
  issue](https://github.com/pls-rs/pls/issues/).

- **Submitting code**

  We welcome your code contributions towards bug fixes, new features and other
  enhancements at our
  [GitHub repository](https://github.com/pls-rs/pls/blob/main/src/).

- **Writing docs**

  If you can improve this documentation, please do so. The docs are
  [co-located with the code](https://github.com/pls-rs/pls/blob/main/docs/)
  inside the `docs/` directory.

- **Starring the repo**

  Be sure to star the [GitHub repo](https://github.com/pls-rs/pls/) as a gesture
  of appreciation if <Pls /> is an improvement to your workflow.

- **Spreading the word**

  If you like <Pls />, you should tell your friends, post about it to social
  media and convince others to try it.

- **Sponsoring me**

  You can [sponsor me](https://github.com/sponsors/dhruvkb), or another
  contributor, to contribute to <Pls /> in any of the ways above.
